## Technical Notes

### Architectural Pivot (Phase 22)

Original plan used MCP semantic search for team summaries. Verification found data structure mismatch - MCP returns unstructured text but UI expects structured data. Gap closure (22-03) created PostgreSQL `team_summaries` table with proper schema:

- `member_id`, `member_name`, `completed_count`, `blocker_count`, `items` JSONB
- TeamSummaryService with CRUD operations
- GET /api/knowledge/insights now reads from PostgreSQL instead of MCP

### External Dependencies

MCP server availability affects code/docs search (returns empty results until server configured). Does NOT break functionality - graceful error handling in place.
